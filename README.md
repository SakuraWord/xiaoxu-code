# xiaoxu-code

端到端开发工作流技能 —— 将 Claude Code 的开发过程结构化为：**需求分析 → 技术方案 → 方案审批 → 开发准备 → 编码实现 → Code Review → 测试**。

## 核心原则

- **方案先行，审批后动工** —— 用户明确批准后才写代码，不凭直觉动手
- **遇疑必问** —— 不确定的事停下来问，不猜测不假设
- **最小改动** —— 只改方案里列出的东西，不做额外"顺手"修改
- **真实代码** —— 方案必须基于实际代码，不凭空编造
- **TDD 驱动** —— 先写测试、看它失败、再写最简实现

## 安装

将本仓库克隆到 Claude Code 的 skills 目录：

```bash
cd ~/.claude/skills
git clone https://github.com/SakuraWord/xiaoxu-code.git
```

## 使用方式

在 Claude Code 会话中，当你提出开发需求（如"开发一个功能"、"实现"、"添加"、"修改"）时，技能会自动触发，按以下流程推进：

### 工作流程

| 阶段 | 说明 |
|------|------|
| 1. 需求分析 | 阅读相关代码，明确功能边界和影响范围，向用户确认理解 |
| 2. 技术方案 | 输出改动文件清单、数据结构、模块拆分、API 定义、数据流 |
| 3. 方案审批 | 提交方案给用户，**等待明确批准后才继续** |
| 4. 开发准备 | 按需创建分支，验证开发环境 |
| 5. 编码实现 | TDD 驱动，先写测试再写代码 |
| 6. Code Review | 派发独立 reviewer 审查逻辑正确性、方案一致性、边界情况 |
| 7. 集成验证 | 运行全部测试，手动验证关键路径 |
| 8. 完成确认 | 汇总改动内容、测试结果、遗留问题 |

## 触发关键词

- 实现 / 添加 / 修改
- 做 / 写
